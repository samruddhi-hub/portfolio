Internship | Healthcare Application

Internship | January, 2020

I worked with an organisation for their healthcare app that connects the user to a real doctor.
As an intern I was given the opportunity to redesign the visual layout of the app's existing interface

Given the existing UI of the healthcare application, I conducted a critical analysis of the current design using various visual and interactive principles, and offered a redesign based on that critique.


Identified Problems:

  • While the 'Select Country' option is unsuitably positioned, it also cannot be accessed easily and is difficult to comprehend at first try.
  • The 'Enter Code' page can be more precise.
  • The 'Help' page is on the right hand corner and seems blank when displayed. Too much white space gives it an empty and disjointed feel.
  • The 'Payment' page has too many elements placed on it. It causes information overload and does not focus on the main task— payment.
  • There is no field 'Enter Name'. The user has to navigate after registering to find the field. Since it is a healthcare application, basic fields need to be asked to be filled out like name, date of birth, blood group, etc.

Potential Fix:

  • I added drop down method to choose user's country which is better placed and in a familiar format.
  • The 'Enter Code' page is much clearer while entering digits.
  • I consolidated the 'Help' page in the 'Menu' tab maintaining familiarity and avoiding white space.
  • The 'Payment' page looks better arranged with fewer and only relevant elements due to hierarchy.
  • I added a 'Enter Name' field on the first page itself. This gives a sense of empathy when the user registers.

This opportunity helped me assimilate how a compelling design can result in a call to action, directly affecting the company’s business goals and revenue.
